# 论文笔记风格指南

## 语言风格
- 用中文写笔记，但保留英文专业术语（如 Attention、Transformer、Channel Independence）
- 像在和同实验室的同学讲解这篇论文，语气自然、有个人观点
- 不要写成教科书或论文综述，要写成**研究者的阅读笔记**
- 用"我"来表达个人思考，如"我认为"、"对我而言"、"我当前更关心的是"
- 段落不要太长，善用 sub-bullets（Tab 缩进的 `-`）来分层展开

## 各字段写法要求

### 摘要 (summary)
- 直接输出论文的**英文摘要原文**，不要自己缩写或改写

### 核心贡献 (tldr / problem / method / result)
- tldr：**只写一句话**，不加粗、不展开
- problem：2-4 个简短 sub-bullets，不要层层嵌套
- method：分模块概述，每个模块一句话
- result：2-3 条核心结论，含关键数字

### 模型架构 (architecture)
- 先给整体思路（3-5 步数据流 + 输入输出形状）
- 再用 `###` 小标题分模块详解
- **每个模块要写充分的解释性文字**，像在给同学讲解：
  - 这个模块为什么这样设计？
  - 它解决了什么问题？
  - 和已有方法有什么区别？
  - 公式的每个符号是什么含义？
- 不要只列公式，要有连贯的解释段落
- 对难理解的概念用 > [!question] callout

### 实验 (experiments)
- 数据集（一句话列出名称）
- Baseline（一句话列出名称）
- 主要结论（2-3 条，每条一行）
- **不要写消融实验细节、不要写实验设置**

### 思考与启发 (thoughts)
- **尽量简洁**，1-2 点即可，也可以留空
- 如果写，要有个人视角（用"我"），点到为止

## 格式规范
- Obsidian callout 语法：`> [!tip]`、`> [!question]`、`> [!quote]`、`> [!warn]`
- 公式用 `$...$`（行内）和 `$$...$$`（独立行）
- 列表用 `-`，子列表用 Tab + `-`
- **绝对不要输出任何文件路径或图片链接**（不要写 `![...](...)` 格式），路径在 Obsidian 中会爆红
- 但在 architecture 和 experiments 中，**应该引用图号**（如"如图4所示"、"Figure 5 展示"），系统会自动插入对应图片
- 对关键架构图和实验结果图，务必要用文字引用图号
- figure_notes 只选最重要的 3-5 张图，不要列全部图片

## 范例：iTransformer 笔记片段

以下是一篇真实笔记的 Problem/Method/Result 和部分 Architecture，作为风格参考：

### Problem 范例
```
- **Problem:** 传统 Transformer 时间序列预测模型通常以时间点为 token，即每个 token 包含同一时间戳下的所有变量。这种方式存在几个问题：
	- 同一时间戳下的变量不一定代表同一个真实事件，可能存在延迟、错位或物理意义差异；
	- 多个变量过早融合到同一个时间 token 中，容易削弱变量自身的独立表征；
	- 时间 token 的感受野局部，可能难以形成稳定的变量中心表示；
	- attention map 可能难以解释，因为它混合了时间依赖和变量混合两类信息。
```

### Method 范例
```
- **Method:** iTransformer 采用倒置视角重新组织输入：
	- 将每个变量的完整历史序列嵌入为一个变量 token；
	- 在变量 token 之间使用 self-attention，用于捕捉多变量相关性；
	- 对每个变量 token 应用 FFN，用于学习该变量自身的非线性时间表示；
	- Transformer 的基础组件基本不变，改变的是组件作用的维度和建模对象。
```

### Architecture 写法范例
```
#### 输入投影：把长时间序列变成 Token

**输入形状**：假设一个多变量序列，变量数为 $N$，回溯窗口长度为 $T$，则原始输入为：
$$X \in \mathbb{R}^{N \times T}$$
**投影操作**：对每个变量 $i$，将其完整的历史序列（长度为 $T$）通过一个可学习的线性层，投影为一个 $d_{model}$ 维的向量：
$$h_i = X_{i,:} \cdot W_{embed}, \quad W_{embed} \in \mathbb{R}^{T \times d_{model}}$$
**关键理解**：
- 这不是「聚合」（如 mean pooling），而是**可学习的嵌入**。
- 时间顺序信息被**结构化地编码**到向量的不同维度中，而不是被丢弃。
```
