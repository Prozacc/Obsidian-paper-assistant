# 论文阅读笔记风格指南

## 写作目标

- 使用中文解释，保留 Transformer、MoE、attention 等英文术语。
- 写成研究者的阅读笔记，不写成百科式综述。
- 优先解释作者为什么这样设计、数据如何流动、每个公式的物理含义。
- 与现有 Obsidian 笔记一致：短段落、清晰层级、必要时使用子列表。

## 固定结构

1. 英文摘要原文。
2. 核心贡献：一句 TL;DR，以及 Problem、Method、Result。
3. 模型/算法架构：全文重点，按模块深入展开。
4. 实验与结果：数据集、baseline、最多三条归纳结论。
5. 个人思考与启发：没有实质内容时写"（待补充）"。
6. 参考文献与链接：只输出确实存在的链接。

## 核心贡献 (TL;DR) 格式

- Problem / Method / Result 用 `- **Label:**` 标签，**只写正文，不重复标签**（模板会统一添加）。
- 子列表统一用 `- `（dash bullet），不用 `1. ` 编号列表。
- 每个 sub-bullet 不超过 2 行，禁止层层嵌套。
- Problem 的子列表示例：`- 同一时间戳下的变量不一定代表同一事件，可能存在延迟或物理意义差异；`

## 架构部分

- 先写 `### 1. 整体思路`，说明核心洞察、输入输出和 3-5 步数据流。
- 后续每个核心模块单独使用三级标题。
- 每个模块遵循：问题语境 -> 具体做法 -> 公式 -> 符号解释 -> 直观含义。
- 用 `**加粗子标题**` 拆分子部分，**不用 `####`**。
- 公式前后必须有解释段落，不能裸堆公式。
- 每个核心模块末尾有 `**关键理解：**` 段落（1-3 条 insight）。
- 图片必须由相邻正文引出，例如"图 2 展示了……"，不能连续堆放图片。
- 不使用问答体，不用 callout 代替正文。
- 架构部分应明显长于核心贡献和实验部分。

## 实验部分格式

- 用行内格式：`- **数据集:** Weather、Traffic、Electricity`
- 不展开为多行列表：避免 `**数据集：**\n- item1\n- item2`
- 最多 3 条主要结论，每条一行定性判断。
- 不罗列完整表格，不写无上下文的指标数字。

## 内容约束

- Problem、Method、Result 字段不要自行重复标签，模板会添加标签。
- 不编造数值、链接、citekey、Zotero URI 或个人观点。
- 实验部分不罗列完整表格，不写无上下文的指标数字。
- `innovations`、`limitations` 等空列表应保持为空，不输出"无"作为孤立 bullet。
- 个人思考必须与用户研究问题有明确关联；无法推断时留待补充。

## Obsidian 格式

- 标题层级连续，不跳级。
- 使用 `$...$` 和 `$$...$$` 书写 LaTeX。
- 图片使用 Obsidian embed，并紧跟在解释该图的句子后。
- frontmatter 的年份标签使用纯年份，例如 `2026`，与既有笔记保持一致。
- 段落之间保持空行分隔，公式和图片前后有空行。
